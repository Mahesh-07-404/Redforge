# SAFETY Skill: Legal Compliance

## Purpose
Understand legal requirements for security testing.

## Computer Laws

### United States

#### CFAA (Computer Fraud and Abuse Act)
```
- Unauthorized access to computers
- Penalties up to 20 years
- Exceeds authorized access = violation
```

#### State Laws
```
- California: Computer Crime Law (Penal Code 502)
- New York: Penal Law Article 156
- Varies by state
```

### European Union

#### NIS2 Directive
```
- Network and information security
- Essential entities must report
```

#### GDPR
```
- Data protection requirements
- Breach notification
- Large fines for violations
```

### Other Countries
```
- UK: Computer Misuse Act 1990
- Canada: Security of Information Act
- Australia: Cybercrime Act
```

## Authorization

### Written Authorization
```markdown
# Authorization Letter Template

Date: [Date]
Tester: [Name/Company]
Organization: [Target Organization]
Scope: [Specific targets and methods]
Duration: [Start - End date]
Contact: [Name, Phone, Email]

Authorization:
This letter confirms that [Organization] authorizes [Tester]
to conduct security testing on [Scope] during [Duration].

Signed: [Authorized Representative]
```

## Safe Harbor

### What It Provides
```
- Protection from prosecution
- Defense against CFAA claims
- Legal safe zone for testing
```

### Requirements
```
- Follow program rules
- Stay within scope
- Report vulnerabilities
- Don't exploit beyond testing
```

## Prohibited Activities

### Never Do
```
1. Access without authorization
2. Exceed scope
3. Cause denial of service
4. Modify systems
5. Steal data
6. Harassment
7. Extortion
8. Public disclosure without notice
```

### Always Avoid
```
- Physical security testing without permission
- Social engineering (usually prohibited)
- Attacks on critical infrastructure
- Testing 911/emergency systems
```

## Documentation

### Keep Records
```
1. Authorization letter
2. Scope documentation
3. Communication records
4. Test methodology
5. Findings and reports
6. Timeline of activities
```

### Timeline
```
- Keep records for 7+ years
- Secure storage
- Access controls
```

## Professional Standards

### Code of Ethics
```
1. Act with integrity
2. Obtain proper authorization
3. Respect privacy
4. Minimize impact
5. Report vulnerabilities responsibly
```

### Certifications
```
- OSCP (Offensive Security)
- CEH (EC-Council)
- CISSP (ISC²)
- GPEN (GIAC)
```

## When in Doubt

### Questions to Ask
```
1. Do I have written authorization?
2. Is this target in scope?
3. Is this technique allowed?
4. What is the time window?
5. Who do I contact for questions?
```

### Actions
```
1. Stop testing
2. Document the question
3. Get written clarification
4. Wait for response
5. Continue only if approved
```

## Consequences of Violations

### Legal
```
- Criminal charges
- Civil lawsuits
- Fines
- Imprisonment
```

### Professional
```
- Loss of certifications
- Industry reputation
- Employment issues
- Bar/licensing problems
```

## Best Practices
```
1. Always get written authorization
2. Document everything
3. Follow program rules exactly
4. Report immediately if issues arise
5. Know local laws
6. Use safe harbor provisions
7. Keep records secure
```
